# Project SpaceSonar X

ID-based ONNX development lab for FPMarkets `US100` `M5` research, export, parity, packaging, and MT5 runtime probes.

Start with `AGENTS.md`. It is the cold-start boot kernel.

For repository indexing and connector-based entry, read `docs/workspace/repo_index.md` next.

Active map:

- `src/spacesonar/`: new reusable Python package surface
- `foundation/`: collectors, features, labels, training, ONNX, parity, MT5, and reusable orchestration
- `configs/`: lab and local runtime configuration templates
- `data/`: generated or local data manifests and snapshots
- `lab/`: hypotheses, campaigns, runs, candidates, and templates
- `runtime/`: ONNX/EA packages and MT5 attempt records
- `docs/workspace/`: current lab truth
- `docs/registers/`: compact indexes
- `docs/contracts/`: data, ONNX, broker, runtime, and handoff contracts
- `docs/agent_control/`: compact routing/control registries

Bulk legacy routing records, external-review records, research archives, cleanup snapshots, old pipeline material, and legacy boot/audit material were deleted by user request. Current work starts from the ONNX lab surfaces above.

This README is a map, not a live status ledger.
