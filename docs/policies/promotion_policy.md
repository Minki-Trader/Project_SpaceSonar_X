# Promotion Policy

Promotion is different from exploration.

## Vocabulary

- `research_candidate`: compute allocation for more experiments.
- `probe_candidate`: target for runtime observation.
- `operating_promotion`: claim that a system should replace or define an operating reference.
- `runtime_probe`: MT5/runtime observation without authority.
- `runtime_authority`: protected claim backed by the required runtime contract and evidence.

## Rules

- Exploration has no promotion gate.
- Promotion gates apply only when making operating, handoff, runtime-authority, economics-pass, live-readiness, reviewed, verified, or pass claims.
- A strong research clue can remain useful without promotion.
- No deleted legacy artifact can supply promotion evidence.
