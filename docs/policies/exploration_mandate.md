# Exploration Mandate

Exploration is open, but input surfaces must be live-observable in the target MT5 environment.

## Core Rule

- Active SpaceSonar research is anchored on `US100` closed `M5` bars.
- Broker-native live-chart auxiliary symbols can be researched and developed when the local FPMarkets MT5 terminal proves real-time chart and bar/tick availability.
- Non-updating symbols, delayed feeds, offline-only data, single-stock symbols without live MT5 charts, `VIX`, and proxy datasets are out of scope.
- No feature set, label, prediction target, direction mapping, holding period, model family, output head, threshold, or risk logic is predefined.
- Gates protect claims, not curiosity. A live-observable idea may be explored freely, but operating meaning still needs evidence.

## Hypothesis Loop

Use this loop for meaningful work:

1. hypothesis
2. task surface definition
3. experiment
4. validation
5. judgment
6. failure or salvage memory

Task surface definition names what the experiment is trying to learn: predict, classify, rank, estimate, detect, filter, size, hold, exit, or diagnose. If that shape is unclear, the run is a scout surface, not a model claim.

MT5 runtime probes are normal evidence when runtime behavior, economics, EA/ONNX meaning, or handoff claims are involved. Do not skip the narrow sufficient probe only because it is expensive.

## Progressive Hardening

- Early exploration can start when missing evidence is named.
- `research_candidate` means compute allocation, not a selected baseline.
- `probe_candidate` means runtime observation target, not runtime authority.
- `runtime_probe` observes runtime behavior; it does not create runtime authority by itself.
- `operating_promotion` and `runtime_authority` require strong matching evidence.
- `promotion_ineligible` is not idea death.

## No Baseline Closure During Exploration

An exploration closeout is a boundary record, not a baseline ceremony.

Allowed closeout words:

- `seed_surface`
- `preserved_clue`
- `reference_surface`
- `negative_memory`
- `invalid_setup`
- `blocked_retry_condition`
- `inconclusive`

Forbidden without a separate promotion or operating record:

- `selected_baseline`
- `operating_reference`
- `promotion_candidate`
- `runtime_authority`
- `economics_pass`
- `live_readiness`

## WFO

Walk-forward optimization is the default serious optimization mode. A single-window read can be used as a scout read only when labeled that way.

## Failure Memory

When an idea fails, record:

- hypothesis
- variants tried
- failed boundary
- why it failed
- salvage value
- reopen condition
- do-not-repeat note

Negative results are useful evidence. Invalid results are not interpreted until the broken assumption is repaired.
